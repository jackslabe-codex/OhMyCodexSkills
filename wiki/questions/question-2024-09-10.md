## Stem

考虑以下 C++ 代码实现的归并排序算法：

```cpp
void merge(int arr[], int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;

    int L[n1], R[n2];

    for (int i = 0; i < n1; i++)
        L[i] = arr[left + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = left;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        }
        else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

void merge_sort(int arr[], int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;

        merge_sort(arr, left, mid);
        merge_sort(arr, mid + 1, right);

        merge(arr, left, mid, right);
    }
}
```

对长度为 $n$ 的数组 `arr`，调用函数 `merge_sort(a, 0, n-1)`，在排序过程中 `merge` 函数的递归调用次数大约是（ ）。

A. $O(1)$

B. $O(n)$

C. $O(\log n)$

D. $O(n\log n)$

## Answer

B

## Analysis

在归并排序中，`merge_sort` 函数的递归调用形成一棵二叉树。对于长度为 $n$ 的数组，递归树的深度为 $\lceil \log_2 n \rceil$。每一层都会对当前子数组调用 `merge` 函数进行合并操作。具体来说，第 $k$ 层有 $2^k$ 个 `merge` 调用。总的 `merge` 调用次数等于内部节点数加上叶子节点的父节点数，约为 $n-1$ 或 $O(n)$ 级别；但若从递归深度角度理解“次数”为层数相关，或理解为 `merge` 被调用的总次数（即所有层合并操作的执行次数），其数量级为 $O(n)$。更准确地说，对于 $n$ 个元素的归并排序，`merge` 函数被调用的次数恰好是 $n-1$ 次（对应于递归树中每个非叶节点一次），因此时间复杂度为 $O(n)$。
