## Stem

硬币找零问题中要求找给客户最少的硬币。coins 存储可用硬币规格，单位为角，假设规格都小于10角，且一定有1角规格。amount 为要找零的金额，约定必须为1角的整数倍。输出为每种规格及其数量，按规格从大到小输出，如果某种规格不必要，则输出为0。下面是其实现代码，相关说法正确的是（ ）。

```cpp
const int MAX_COINS = 10;
int result[MAX_COINS] = {0}; // 假设最多10种面额

int find_coins(const vector<int>& coins, int amount) {
    sort(coins.begin(), coins.end(), greater<int>());

    int n = coins.size();

    for (int i = 0; i < n; ++i) {
        int coin = coins[i];
        int num = amount / coin;
        result[i] = num;
        amount -= num * coin;
        if (amount == 0) break;
    }

    cout << "找零方案如下：" << endl;
    for (int i = 0; i < n; ++i) {
        cout << sorted_coins[i] << "角需要" << result[i] << "枚" << endl;
    }

    return 0;
}
```

A. 上述代码采用贪心算法实现
B. 针对本题具体要求，上述代码总能找到最优解
C. 上述代码采用枚举算法
D. 上述代码采用分治算法

## Answer

A

## Analysis

该代码先对硬币面额从大到小排序，然后依次尝试使用尽可能多的大面额硬币，这是典型的贪心策略。对于硬币找零问题，贪心算法并不总能得到最优解（如硬币面额为{1,3,4}要找6时，贪心得4+1+1共3枚，而最优为3+3共2枚），但在某些特定面额系统（如包含1且满足特定条件）下可以得到最优解。该代码明显不是枚举（未穷举所有组合）也不是分治（无分解子问题）。
