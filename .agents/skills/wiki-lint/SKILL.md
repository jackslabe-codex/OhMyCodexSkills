# wiki-lint 技能：知识图谱健康巡检

## 核心目标
将软件工程中的“静态代码分析”引入知识管理。定期运行本 skill，找出知识库长期演进中产生的结构问题，例如死链、孤岛和未同步索引。

## 前置依赖

本 skill 假设 wiki 已完成基础初始化。若发现以下任一项缺失，先执行 `wiki-init` 技能，再继续 lint：

- `wiki/`
- `wiki/index.md`
- `wiki/log.md`

## 触发条件
- 用户输入 `/lint`
- 用户询问“我的知识库健康状况如何”
- 用户要求“检查知识库状态”或“检查健康”

## 巡检流水线

### 第 1 步：索引一致性检查
1. 读取 `wiki/index.md` 全部内容
2. 扫描 `wiki/` 下所有内容页面，排除系统文件 `index.md`、`log.md`、`tags.md`
3. 提取 index.md 中注册的所有双链链接 `[[页面名称]]`
4. 比对：找出已注册但文件不存在的条目 OR 文件存在但未注册的页面

### 第 2 步：双向链接健康检查
1. 扫描所有内容页面，提取所有 `[[双链]]` 格式的链接
2. 如果链接指向的页面不存在 → 标记为**死链**
3. 统计被引用的页面（排除 self-reference）
4. 找出从未被任何其他页面引用的页面 → **孤儿页面**

### 第 3 步：系统文件一致性检查
1. 检查 `wiki/index.md` 是否存在四个主分区：`Sources`、`Entities`、`Concepts`、`Syntheses`
2. 检查 `wiki/tags.md` 是否存在，并可被读取为标签词表
3. 若系统文件缺失、为空或结构明显损坏，在报告中单独标记

## 报告输出规范

扫描完成后，输出结构化报告，遵循以下格式：

```markdown
## 知识库健康检查报告 — YYYY-MM-DD

### 通过项
- [运行良好的项目]

### 关注项
- **发现 N 个孤儿页面**：[列表]
- **发现 N 个未同步索引页面**：[列表]

### 问题项
- **发现 N 个死链**：[来源页面] → [[不存在的目标页面]]
- **系统文件异常**：[异常文件列表]

### 建议动作
- [最值得优先修复的问题]
```

## 硬约束
- **仅读扫描**：生成报告前，禁止修改、删除、重命名任何文件
- **手动确认**：报告后等待用户确认再执行修复
- **不默认写日志**：普通 lint 扫描不应自动写入 `wiki/log.md`
- **不扫描 raw/**：本 skill 只检查 wiki 结构，不检查收件箱积压
